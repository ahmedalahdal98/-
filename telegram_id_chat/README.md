# محادثة تيليجرام بواسطة ID

أداة بايثون صغيرة تستخدم **حساب تيليجرام الشخصي** لمراسلة مستخدم سبق أن
ظهر للحساب، اعتمادًا على رقم الـID. لا تستخدم Bot Token.

## الإعداد

1. استخرج `api_id` و`api_hash` لحسابك من <https://my.telegram.org> ثم اختر
   **API development tools**.
2. افتح الملف `private_config.py` وضع القيم الثلاث في بدايته:

   ```python
   API_ID = 12345678
   API_HASH = "ضع_api_hash_الحقيقي_هنا"
   TARGET_USER_ID = 123456789
   ```

   `API_ID` هو رقم التطبيق، و`TARGET_USER_ID` هو آيدي الشخص. لا تضع علامات
   تنصيص حول الرقمين.
3. افتح PowerShell داخل هذا المجلد وثبّت المكتبة:

   ```powershell
   py -m pip install -r requirements.txt
   ```

## التشغيل

لفتح محادثة تفاعلية مع الشخص المحدد في `private_config.py`:

```powershell
py .\chat_by_id.py
```

للتأكد من الحساب المسجل وعرض آيديات جميع المحادثات الخاصة المتاحة:

```powershell
py .\chat_by_id.py --list-users
```

لإرسال رسالة واحدة ثم الخروج:

```powershell
py .\chat_by_id.py --message "السلام عليكم"
```

في أول تشغيل سيطلب تيليجرام رقم هاتفك ورمز تسجيل الدخول، وقد يطلب كلمة مرور
التحقق بخطوتين. بعد ذلك يعرض البرنامج اسم المستلم وID ويطلب منك كتابة `نعم`
قبل الإرسال.

## حدود مهمة

- الـID وحده لا يكفي إذا لم يستطع Telethon إيجاد الشخص في محادثات الحساب أو
  في ذاكرة ملف الجلسة أو بين أعضاء القروبات المتاحة؛ تيليجرام يحتاج أيضًا إلى
  `access_hash` غير الظاهر لك.
- إذا لم يجد البرنامج الآيدي في الخاص، يبحث تلقائيًا بين أعضاء القروبات التي
  يسمح تيليجرام للحساب بعرض قائمة أعضائها. قد يستغرق ذلك وقتًا مع القروبات الكبيرة.
- إذا حذفت المحادثة ولم يعد الحساب قادرًا على الوصول إلى المستخدم، استخدم
  اسم المستخدم `@username` أو افتح المحادثة مرة أخرى أولًا.
- ملف `telegram_user.session` حساس جدًا ويعادل جلسة دخول؛ لا ترسله لأي شخص.
- استخدم الأداة للمراسلات المشروعة فقط، ولا تستخدمها للإزعاج أو الإرسال الجماعي.

> إذا كنت تقصد **بوت تيليجرام** بدل حسابك الشخصي، فالبوت لا يستطيع بدء محادثة
> خاصة مع مستخدم لم يرسل له رسالة أولًا.
